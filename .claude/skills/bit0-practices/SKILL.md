# bit0-armbian coding practices

These rules come from the 2026-07 replicability audit (`audit.md` — a local,
gitignored working doc; never commit it). Apply them to every change; when a
change lands that resolves an audit finding, update the local `audit.md`
status lines alongside it.

## Replicability (the project's #1 goal)

- **No machine-specific absolute paths.** Never default to a personal path
  (`/home/<user>/...`). Scripts take required arguments or documented env vars;
  repo-internal paths are resolved relative to `BASH_SOURCE`/`__file__`.
  Absolute paths are correct only for *target-filesystem* locations that ship
  in the image (`/usr/local/bin/...`, `/etc/...`).
- **Never commit a binary that has source in this repo.** Anything buildable
  (`.so` shims, panel firmware `.bin`) must be generated during the image build
  — compile inside the chroot in `customize-image.sh` (install toolchain, build,
  purge toolchain). Keep sources where the chroot can see them
  (`userpatches/overlay/...`, bind-mounted at `/tmp/overlay`).
- **Third-party blobs that can't be built** (e.g. the transplanted `adbd`) need:
  documented origin + license in `docs/BINARIES.md`, and a `sha256sum` guard in
  `customize-image.sh` that fails the build on mismatch.
- **Proprietary files are never committed.** PICO-8 lives only in the gitignored
  `userpatches/overlay/root/pico-8/`; build and launch logic must degrade
  gracefully when it's absent (the `PICO-8 NOT INSTALLED` pattern).
- **Gitignore autogenerated files** (e.g. `userpatches/config-example.conf`);
  don't commit framework-generated content.
- **No per-unit data as image defaults.** Device-specific calibration or IDs
  must not be baked into the overlay; derive per-unit values at runtime
  (e.g. USB serial from `/proc/device-tree/serial-number`).

## Shell scripts

- Start with `set -euo pipefail` (`customize-image.sh` included — it runs
  standalone in the chroot). Annotate intentionally-fallible commands with
  explicit `|| true` or a check; never rely on silent failure.
- Image hooks fail **loudly**: if a required artifact is missing, `exit 1` with
  a `bit0: FATAL:` message — a bootable-but-broken image is the worst outcome.

## Python daemons (`userpatches/overlay/usr/local/bin/`)

- **Stdlib only** (plus `python3-serial` for the UART bridge). The target has
  128 MB RAM — no new runtime dependencies without strong justification.
- **Shared plumbing goes in `userpatches/overlay/usr/local/lib/bit0/`** (evdev
  constants/IO, uinput device creation, fb size probe, `find_event`). Never
  copy-paste these between daemons; import via
  `sys.path.insert(0, '/usr/local/lib')`.
- **Resolve input devices by name** (`/sys/class/input/*/device/name`), never by
  hardcoded `/dev/input/eventN` — node ordering is not stable across boots.
- **Subprocess in list form.** No `os.system()`, no `shell=True`. Even static
  command strings get converted when touched — the pattern must not survive to
  the day a filename gets interpolated.
- **One source of truth per action.** Volume, brightness, power, etc. each have
  a single implementation (helper script or lib function) called from every
  trigger path (launcher UI, UART bridge, triggerhappy). No duplicated `amixer`
  incantations; delete dead trigger configs instead of leaving them "just in case".
- **Feature entries must degrade gracefully**: a launcher menu item whose
  binary/asset is missing either hides itself or shows a clear on-screen
  message — never a silent exception-swallowed bounce back to the menu.
- New scripts use a `main()` entry point (no work at import scope) and release
  grabbed devices / restore state in `finally`.

## Launcher UI specifically

- The redraw choreography (compose-then-single-flush, cursor erase/draw order,
  post-`openvt` repaint delay) is hard-won against real hardware artifacts.
  Don't reorder drawing or flushing logic without an on-device test.
- Removed features are removed completely: constants, functions, page handlers,
  `_kill_stale` entries, and every doc mention — grep the repo for the feature
  name before finishing.

## systemd units

- Add `NoNewPrivileges=yes` to new units; `ProtectHome=yes` where the daemon
  doesn't touch `/root`. Root + no sandbox is acceptable only where fb0/uinput/
  sysfs access requires it.
- Overridable hardware identity (UART device, input-device names) goes in unit
  `Environment=` lines, read with `os.environ.get(..., default)` — behavior
  tunables stay as code constants.

## Verification before "done"

- Python: syntax-check every edited script (`python3 -m py_compile` or
  `ast.parse`).
- Shell: `bash -n`, and shellcheck if available.
- Grep for stale references (feature names, line-number citations in docs)
  after any removal or file restructure — including `audit.md` and
  `docs/PORTING-NOTES.md`.
- Changes to daemons, units, or the launcher need an on-device smoke test noted
  in the commit/PR description (keyboard, touch, audio, and the affected flow).
